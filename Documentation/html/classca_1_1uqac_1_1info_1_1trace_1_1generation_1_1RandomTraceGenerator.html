Detailed Description

Randomly produces traces of events based on a number of parameters. More precisely, the RandomTraceGenerator takes as parameters:

  1. The interval for the number of messages to produce
  2. The maximum arity of each message
  3. The number of available parameter names
  4. The maximum domain size for all parameters
  5. Whether each event can be multi-valued or not

Calls to generate produce message traces where values for each parameter are randomly chosen from a uniform distribution in the interval provided.

void ca.uqac.info.trace.generation.RandomTraceGenerator.setNumParameters ( int  n)

Specifies the number of available parameters to choose from when building an event in the output trace.

Caveat emptor: you will run into a problem if num_parameters < max_arity, and multivalued = false. This effectively tells the generator to produce messages with a number of param-value pairs greater than the number of available parameters, yet forbids it to repeat parameters. No check is made about that in the code.
