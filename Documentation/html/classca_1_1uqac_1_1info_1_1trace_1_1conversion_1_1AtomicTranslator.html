Detailed Description

Converts an arbitrary trace and formula to an atomic (i.e. propositional) problem.

The AtomicTranslator takes as input an arbitrary event trace and and converts it to an atomic trace &sigma', where every event of the trace becomes a single ("atomic") symbol.

The translator does the same thing with the formula , yielding a formula ' where every ground term is replaced by an atom (or a disjunction of atoms). The translator does so in such a way that  &#8871;  if and only if ' &#8871; '.

Note that translateFormula must be called after translateTrace, as it reuses information from reading the trace to generate the formula.

Example usage

Consider the following trace of four events: 

 <Event>
   <a>1</a>
   <b>2</b>
   <c>3</c>
 </Event>
 <Event>
   <a>2</a>
   <b>6</b>
   <c>1</c>
 </Event>
  <Event>
   <a>1</a>
   <b>3</b>
   <c>1</c>
 </Event>
  <Event>
   <a>2</a>
   <b>6</b>
   <c>4</c>
 </Event>

Suppose we want to create an atomic trace that distinguishes the values of parameters a and b.

  1. One first calls translator.setParameters(s), where s is a set containing a single String "a".
  2. One then calls translator.translateTrace(t), where t is the trace shown above. The resulting trace will be <blockquote> e0 e1 e2 e1 </blockquote> that is, the 2nd and 4th events have the same token, since they have the same combination of values for parameters a and b (and we discard the other parameters).
  3. Suppose we want to convert the LTL formula F a=1. One calls translator.translateFormula(f), where f is an Operator containing the formula. The method returns the string <blockquote> F (e0 | e2) </blockquote> that is, the condition a=1 is fulfilled by an event if its token is either e0 or e2.

One can see how a problem on a trace/formula with parameters has been converted into a situation where both the trace and the formula contain only atomic events.

String ca.uqac.info.trace.conversion.AtomicTranslator.createTokenFromEvent ( Event  e) [protected]

Creates an "atomic" token from an arbitrary event. The process goes as follows: given an event of the following form: 

 <Event>
   <p0>123</p0>
   <p1>456</p1>
   <p2>789</p2>
 </Event>

Suppose the user has provided p0 and p1 as the parameters to take into account when forming the atomic symbol. The tokenizer will create an event token p0_123_p1_456.

Parameters:
eThe event to process
Returns:
A string rendition of the atomic token
